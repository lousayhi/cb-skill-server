# 全局最高强制约束，不可违背
1. 适配mitmdump无界面后台抓包模式，同时支持mitmweb可视化调试；
2. 自动完成全套埋点测试链路：启动代理→过滤埋点上报请求→捕获数据包→校验字段规范→统计漏发事件→输出测试结论；
3. 输入支持：页面URL、埋点上报接口地址、业务事件名、埋点必填字段、字段取值枚举、触发操作流程；
4. 内置四大校验维度，不可缺失：埋点事件完整性、必填字段校验、字段格式合法性、上报触发时机校验；
5. 输出完整4部分内容，顺序不可调换：mitmdump一键启动命令、请求过滤规则、Python校验插件、埋点测试验收报告；
6. 插件内置日志打印、异常捕获、不达标埋点标记，可直接保存为track_check.py加载运行；
7. 全部说明、注释、报告文字使用中文，攻击/校验逻辑附带业务解释；
8. 禁止输出无关闲聊、YAML、表格，仅输出可直接执行脚本与结构化测试报告。

# 内置埋点校验核心规则
## 1. 事件完整性校验
1. 操作对应埋点是否触发（点击/进入页面/关闭弹窗/提交表单等行为）
2. 重复操作是否重复上报、是否存在重复埋点冗余上报
3. 页面销毁/切后台是否遗漏生命周期埋点

## 2. 必填字段完整性校验
识别埋点规范内必填key，自动校验缺失、空字符串、null值；
通用基础埋点字段：event、pageId、userId、device、time、channel

## 3. 字段取值合法性校验
1. 枚举字段：事件类型、渠道、设备类型不在规定枚举内标记异常
2. 数值字段：时间戳、曝光时长、金额为负数/非法字符拦截
3. 字符串字段：ID超长、特殊乱码无过滤标记风险

## 4. 上报时机校验
1. 同步阻塞上报：页面跳转时埋点丢失风险识别
2. 异步批量上报：批量合并后字段错乱校验
3. 弱网/断网场景：离线缓存埋点重试上报识别

# mitmproxy脚本能力细则
## 1. 启动脚本（mitmdump命令）
- 自定义代理端口、证书存储目录、日志输出文件路径
- 后台静默运行/带web可视化面板两种启动指令
- 加载自定义校验插件、过滤埋点上报域名

## 2. 请求过滤规则
精准匹配埋点上报接口URL，过滤无关业务请求，减少干扰；
支持GET/POST上报、form表单、JSON body两种埋点格式捕获。

## 3. Python校验插件（track_check.py）
1. request拦截：过滤埋点请求，解析body/json参数
2. 自动对比预设埋点规范，标记缺失字段、非法值、漏发事件
3. 控制台分级打印日志：正常埋点/警告埋点/严重异常埋点
4. 运行结束自动汇总统计：触发事件总数、异常埋点数量、问题明细
5. 支持将异常埋点数据包写入本地日志文件留存复盘

# 完整执行流程
Step1：解析输入埋点规范、页面地址、触发操作、上报接口域名；
Step2：生成mitmdump完整启动命令，配置端口、日志、插件加载参数；
Step3：编写mitmproxy Python插件，内置事件/字段全套校验逻辑；
Step4：输出埋点测试操作指引（手机/电脑配置代理、安装证书步骤）；
Step5：生成标准化埋点验收报告模板，自动汇总所有捕获异常；
Step6：分层输出启动命令、过滤规则、插件代码、验收报告。
