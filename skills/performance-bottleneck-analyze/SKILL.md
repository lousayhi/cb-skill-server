# 内置瓶颈自动识别判定规则
## 1. 数据库索引瓶颈
判定特征：慢日志大量全表扫描、explain type=ALL、TPS低、RT随数据量线性上涨、offset超大分页耗时暴增；
根因：无索引、联合索引不满足最左匹配、字段隐式转换、索引区分度低；
优化：新增合适索引、改写查询条件、分页优化、避免索引失效语法。

## 2. 接口串行执行瓶颈
判定特征：单接口RT极高、TPS上不去、无CPU/DB压力，接口内部多同步串行调用第三方/DB；
根因：同步循环查询、串行调用多个下游接口、无异步多线程处理；
优化：并行调用、异步线程池、批量聚合查询。

## 3. 锁竞争瓶颈（DB锁/分布式锁）
判定特征：慢日志大量lock wait超时、并发升高后TPS暴跌、RT抖动剧烈；
根因：长事务、大事务批量更新、行锁升级表锁、分布式锁持有时间过长；
优化：拆分短事务、更新缩小范围、减少锁持有时长、分段更新。

## 4. 磁盘IO瓶颈
判定特征：iowait持续高于70%、磁盘读写打满、数据库刷脏页频繁、RT缓慢上涨；
根因：大批量写入、无批量提交、日志同步刷盘、大量临时文件读写；
优化：批量操作、调整刷盘策略、冷热数据分库、减少临时文件。

## 5. JVM内存/GC瓶颈
判定特征：CPU高、频繁Full GC、内存持续上涨、接口偶发超时；
根因：大对象未释放、内存泄漏、堆内存分配不足、频繁创建销毁对象；
优化：调优堆参数、复用对象、释放大集合、优化GC回收频率。

## 6. 网络瓶颈
判定特征：跨机房调用、大报文传输、RT波动大，本地压测正常线上性能差；
根因：单次传输报文过大、频繁跨机房请求、无连接池复用；
优化：报文压缩、连接池复用、批量合并请求、就近部署服务。

# 完整执行流程
Step1：读取全部输入性能指标、慢日志、资源监控数据；
Step2：按指标阈值匹配内置瓶颈判定规则，汇总全部瓶颈点；
Step3：分层拆解每个瓶颈：表面现象→底层根因；
Step4：输出研发侧可落地优化改造方案；
Step5：配套输出优化后的验证压测方案（并发、指标阈值、观测维度）；
Step6：整合四段式结构化中文报告输出。

# 固定输出四段结构（严格遵守顺序）
## 一、性能瓶颈总览
汇总当前系统存在的全部瓶颈，标注严重等级：严重/一般/轻微，简述整体性能现状。
## 二、分维度根因详细拆解
逐个瓶颈说明：观测现象、指标依据、底层技术根因。
## 三、分模块优化落地方案
1. 数据库优化方案
2. 代码逻辑优化方案
3. 服务器/中间件资源优化方案
## 四、优化后验证测试方案
明确压测并发、观测指标、验收阈值、对比基准场景，用于验证优化效果。
