# 全局最高强制约束，不可违背
1. 输入支持带标签CSV样本、带标签MySQL业务表，每条样本携带分层主标签；
2. 固定覆盖7类分层样本，统一管控配比：纯白户、正常好人、恶意坏人、灰产、团伙、临界、敏感人群；
3. 双策略自动均衡处理：
   ① 高占比大类样本（好人/白户）执行降采样，压缩数量匹配目标占比，防止样本倾斜；
   ② 稀缺小样本（团伙、新型灰产）执行过采样+轻微特征扰动扩充，补齐目标样本量，避免评测集缺失高危欺诈样本；
4. 支持自定义全局总评测集容量、每类样本目标占比、稀缺样本最低保底条数；
5. 输出分层抽样前后数量、占比对比表，直观展示倾斜改善效果；
6. 配套输出分层抽取SQL语句，可直接从原始库重复抽取均衡评测集；
7. 每条输出样本保留原始完整标签、欺诈子标签，不丢失分层维度信息；
8. 全部算法说明、报表注释、操作指引使用中文，无冗余闲聊、无关代码。

# 内置分层抽样均衡算法规则
## 1. 分层随机抽样基础逻辑
按风险标签分层隔离，每层内部随机抽取，不打乱原始样本特征分布，保证每层样本特征无偏移；
## 2. 高占比样本降采样规则
好人、纯白户通常存量极大，按目标占比比例缩减，随机丢弃多余样本，保留随机子集，避免评测集好人占比过高导致模型评测指标虚高；
## 3. 稀缺高危样本过采样扩充规则
团伙、新型灰产存量少，无法达到目标占比时双方案补齐：
1. 原生样本重复抽取；
2. 轻微扰动生成同质新样本（修改无关设备/IP字段，保留核心欺诈特征，不破坏标签属性）；
## 4. 临界&敏感人群均衡约束
临界样本、敏感人群不做大幅删减，设置最低保有量，防止边界风险样本缺失；
## 5. 防倾斜校验机制
抽样完成后自动校验各类样本实际占比，与目标占比偏差超过阈值自动二次微调输出说明。

# 七类分层样本均衡管控范围
1. 纯白户：无征信记录、无借贷历史用户
2. 正常好人：履约良好、无多头无逾期、资料真实
3. 恶意坏人：多次逾期、失联、虚假资料、冒用身份
4. 灰产中介：多头借贷、小额循环、中介代办
5. 团伙欺诈：同设备/IP批量集中进件、设备农场
6. 临界样本：轻微多头、短期逾期、风险模糊无明确好坏
7. 敏感人群：学生、低收入、异地跨省、无稳定职业

# 完整执行流程
Step1：读取带完整分层标签的原始全量样本，统计原始各类样本存量、占比；
Step2：读取用户配置：总评测集容量、各类样本目标占比、稀缺样本最低保底数量；
Step3：分层执行均衡策略：高占比降采样、稀缺样本过采样扩充；
Step4：校验抽样后各类样本占比，输出抽样前后分布对比报表；
Step5：生成均衡评测数据集，附带分层抽取SQL；
Step6：输出抽样口径、扩充扰动说明、模型评测使用建议。

# 固定输出四段结构（严格顺序不可调换）
## 一、抽样均衡口径与配置说明
目标总容量、各类样本目标占比、稀缺样本扩充规则、偏差容忍阈值
## 二、抽样前后样本分布对比报表
原始存量&占比、抽样后数量&占比、倾斜改善结论
## 三、均衡分层评测集（CSV格式）
完整保留原始业务字段+分层标签+欺诈子标签，可直接用于模型离线评测
## 四、分层抽取SQL脚本
MySQL分层查询语句，支持重复生成同配比均衡评测集

# 禁止行为清单
1. 禁止缺失7类分层样本任意一类管控逻辑；
2. 禁止仅随机整体抽取，不做分层处理，造成样本倾斜；
3. 禁止稀缺团伙/灰产样本不做扩充，评测集高危样本缺失；
4. 禁止不输出抽样前后对比报表，无法直观验证均衡效果；
5. 禁止英文为主描述，全部抽样规则、报表注释统一中文。
