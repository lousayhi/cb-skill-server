# 全局最高强制约束，不可修改
1. 输入支持基线快照数据集、待上线新版快照数据集、样本真实逾期标签，通过sample_id关联打分与好坏标签；
2. 一次性批量计算完整风控模型核心指标，不可缺失任一核心指标：
   AUC、KS、各分箱KS、TOP10%/TOP20%坏样本捕获率、各分箱坏账率、整体通过率、整体坏账率、LIFT提升度、Precision、Recall、F1
3. 支持多维度分层拆解指标：全量汇总、风险分层样本、模型分箱分段、渠道、地域、年龄、收入人群；
4. 自动实现新旧版本指标差值对比，自定义指标恶化阈值，下滑超阈值自动标记高风险；
5. 分层两级输出：全局大盘指标、细分人群分层指标，定位哪类群体模型效果衰减；
6. 自动关联样本快照数据，输出指标下滑对应的可疑样本明细查询SQL，用于定位根因；
7. 风险分级：小幅波动、中度衰减、高危显著恶化三级标注，给出上线决策建议；
8. 全部指标释义、分层说明、风险提示统一中文，无多余闲聊、无关冗余代码。

# 内置模型指标批量计算规则
## 1. 全局大盘核心指标
1. AUC：区分好坏样本整体能力
2. KS：好坏样本最大分离度，衡量模型区分能力
3. 整体通过率：模型策略下放款占比
4. 整体坏账率：放款样本中逾期坏样本占比
5. LIFT：相比随机筛选，模型捕获坏样本提升倍数
6. Precision/Recall/F1：固定阈值下精准率、召回率、综合指标

## 2. 分箱分层指标
按模型分等宽/等频分箱，每箱输出：样本量、占比、坏账率、累计坏占比、单箱KS；
自动识别高分箱坏账上涨、低分箱坏账异常等退化场景。

## 3. TOP段捕获指标
TOP10%、TOP20%高分段坏样本覆盖率，衡量模型前置拦截能力。

## 4. 多维度分层拆分规则
按以下维度独立重算全套指标，定位局部衰减：
- 风险分层：好人/坏人/灰产/团伙/临界/敏感人群
- 人群维度：年龄、收入、地域、申请渠道
- 欺诈类型：代办、设备农场、虚假资料、多头借贷

## 5. 新旧版本对比判定规则
用户自定义各指标波动阈值（KS下降、AUC下降、坏账上升、LIFT下降），超出阈值标记指标恶化；
区分：整体大盘恶化、仅某个人群分层恶化、仅某分箱区间恶化。

# 完整执行流程
Step1：读取基线快照、新版快照、样本逾期真实标签，通过sample_id关联模型分与好坏标识
Step2：批量计算新旧两套模型全局全量指标，同步按分层维度拆分指标
Step3：对比新旧指标差值，匹配阈值标记衰减/恶化指标，标注风险等级
Step4：汇总全局对比报表、分层人群明细、分箱坏账明细
Step5：生成指标下滑样本排查SQL，可筛选对应分层/分箱样本复现问题
Step6：输出模型上线综合评估结论与优化建议

# 固定输出四段结构（严格顺序不可调换）
## 一、全局模型指标新旧对比总表
全套指标新旧数值、差值、是否恶化、风险等级、波动幅度
## 二、分层人群指标明细报表
按风险分层/渠道/年龄/地域拆分各群体全套模型指标，定位局部衰减群体
## 三、模型分箱坏账分布明细
新旧版本各分箱样本量、坏账率、累计坏占比对比，识别分箱漂移
## 四、指标恶化样本排查SQL
按衰减维度筛选异常样本，用于特征/规则根因排查

# 禁止行为清单
1. 禁止缺失AUC、KS、分箱坏账、TOP捕获率、LIFT等核心风控指标；
2. 禁止仅输出全局指标，不做分层、分箱拆解，无法定位局部衰减；
3. 禁止不支持新旧版本横向对比，无指标恶化标记；
4. 禁止不输出分层明细与排查SQL，仅提供汇总数据；
5. 禁止英文为主描述，指标解释、风险判定、评估结论统一中文。
