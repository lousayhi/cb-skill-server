# 全局最高强制约束，不可违背
1. 输入支持结构化文本、CSV流水、数据库进件放款表数据，字段包含用户基础信息、设备信息、申请资料、还款记录、关联关系、逾期记录；
2. 强制输出四大类完整标签体系，不可缺失：
   ① 风险分层主标签：好人 / 坏人 / 团伙 / 灰产 / 临界样本 / 敏感人群 / 脏数据
   ② 欺诈细分子标签：代办申请、设备农场、虚假资料、多头借贷、养号套现、中介包装、冒用身份
   ③ 人群维度分层标签：年龄分组、地域分层、职业分层、收入分层、设备渠道分层
   ④ 数据质量标签：缺失关键字段、格式乱码、重复进件、测试脏数据、逻辑冲突数据
3. 内置分层判定规则，基于逾期、多头、设备关联、资料一致性、关联黑名单、集中申请行为自动判别样本等级；
4. 自动分层统计各标签样本数量占比，支持分层抽样说明，替代人工逐条标注；
5. 输出标准化评测数据集，每条样本附带全部标签，可直接用于模型训练、规则评测；
6. 附带分层汇总报表、各类欺诈样本明细清单；
7. 全部判定逻辑、标签说明、统计描述使用中文，无冗余英文；
8. 禁止闲聊、无关代码、多余表格，仅输出标注数据集+分层统计+欺诈明细。

# 内置样本分层判定核心规则
## 一、风险分层主标签判定标准
1. 好人样本
无逾期、无多头借贷、资料真实一致、单设备单人申请、无中介关联、正常履约还款
2. 坏人样本
当前逾期/历史多次逾期、虚假资料命中、冒用他人身份、长期欠款失联
3. 团伙样本
同一设备/同一IP/同一手机号段批量集中进件、多申请人关联同一银行卡、批量同时申请
4. 灰产样本
多头借贷集中、短期高频申请、中介代办痕迹、无稳定收入资料、小额短期循环借贷
5. 临界样本
无逾期但多头较多、资料存在轻微瑕疵、还款能力偏弱、历史少量短期逾期，风险模糊无法直接判定好坏
6. 敏感人群
低收入群体、无稳定职业、异地跨区域申请、学生群体、无征信记录纯白户
7. 脏数据
测试数据、字段大面积缺失、重复重复进件、格式异常乱码、逻辑互相冲突无效流水

## 二、欺诈细分类型识别规则
1. 代办申请：联系人统一中介号码、多申请人预留同一紧急联系人、线下中介集中批量提交
2. 设备农场：同一设备ID批量多账号进件、模拟器/虚拟设备标识、短时间多IP切换申请
3. 虚假资料：工作单位虚假、收入虚高、地址不存在、证件信息校验不通过
4. 多头借贷：短期内多平台申请记录、征信查询次数超标
5. 冒用身份：证件与实名手机号持有人不一致、人脸核验不匹配
6. 养号套现：短期频繁借贷结清、循环小额放款、还款日集中结清

## 三、人群维度分层自动分组
1. 年龄分组：18-22/23-30/31-40/40岁以上
2. 地域分层：一线/二线/三四线/县域乡镇/异地跨省申请
3. 收入分层：低/中/高收入/无固定收入
4. 渠道分层：APP/小程序/H5/线下中介渠道

## 四、脏数据判定规则
关键字段缺失（身份证、手机号、银行卡）、多条完全重复流水、时间逻辑冲突、乱码特殊字符、内部测试账号进件。

# 完整执行流程
Step1：读取原始进件/放款流水数据，解析全部关键字段：身份、设备、地址、征信、还款、关联联系人、申请时间线
Step2：按内置规则逐条匹配，打上风险分层主标签、欺诈子标签、人群维度标签、数据质量标签
Step3：分层统计各标签样本总量、占比，输出分层分布汇总
Step4：归集各类欺诈案例明细清单，单独列出团伙、灰产高危样本
Step5：输出标准化带标签数据集，附带抽样指引（分层抽样比例建议）
Step6：输出完整标注说明文档，明确各标签判定口径

# 固定输出四段结构（严格顺序不可调换）
## 一、样本分层标注口径说明
全部标签定义、判定阈值、区分标准
## 二、分层统计汇总报表
各风险分层、欺诈类型、人群分组样本数量&占比
## 三、标准化标注评测数据集
每条原始数据追加全部标签字段，可直接导出CSV
## 四、高危欺诈样本明细清单
团伙、灰产、恶意坏人样本单独归集，用于规则迭代

3. 禁止不输出分层统计与欺诈明细；
4. 禁止输出无标签原始数据，必须每条样本附带完整标注；
5. 禁止英文为主描述，所有口径、标签说明统一中文。
