# 全局最高强制约束，不可违背
1. 输入支持均衡分层评测集、基线版本信息、特征清单、规则列表、模型列表、分流决策逻辑；
2. 强制五层完整链路快照采集，不可缺失任意一层数据：
   ① 原始样本基础快照：用户身份、设备、申请流水、分层风险标签、欺诈子标签
   ② 特征层快照：全部衍生特征值、特征缺失标识、特征加工异常标记
   ③ 规则引擎快照：命中规则ID、规则名称、规则阈值、命中结果、拦截权重
   ④ 模型层快照：各子模型分数、综合模型分、分箱区间、模型版本标识
   ⑤ 决策结果快照：分流通道、审批策略、最终放款/拒绝/人工复核结果、拒绝原因码
3. 每条快照绑定唯一基线版本号、样本唯一ID，用于多版本横向对比评测；
4. 自动生成快照专用存储表CREATE TABLE语句，字段覆盖五层快照全部维度，适配MySQL存储；
5. 根据评测集批量生成分片INSERT入库SQL，单批次可控，避免超大事务；
6. 内置快照一致性校验逻辑：样本ID唯一、版本隔离、五层数据一一对应；
7. 输出分层结构化内容，无闲聊、无关冗余代码，所有注释、口径说明统一中文；
8. 支持多基线版本批量快照灌入，区分历史基线与新版基线，方便效果对比。

# 五层链路快照采集内置规则
## 1. 原始样本基础快照层
存储字段：样本唯一ID、基线版本号、风险分层标签、欺诈类型标签、用户基础信息、设备信息、申请时间、渠道、原始流水完整字段；
作用：留存原始样本，用于后续复现、回溯、分层抽样校验。

## 2. 特征加工快照层
存储字段：全部衍生特征key-value、特征计算耗时、特征缺失标记、极值/脏数据标记、特征加工异常日志；
作用：定位特征漂移、特征加工bug、异常特征对决策的影响。

## 3. 风控规则命中快照层
存储字段：命中规则ID、规则名称、规则阈值、判断逻辑、是否触发拦截、规则风险等级、批量命中清单；
作用：基线规则拦截效果量化，对比新版本规则命中差异。

## 4. 模型打分快照层
存储字段：各子模型分数、融合综合分、分数分箱、模型版本、模型入参特征列表、模型推理耗时；
作用：对比新旧模型分数分布、区分样本模型分偏移程度。

## 5. 分流决策结果快照层
存储字段：分流通道ID、审批策略、最终决策（放款/拒贷/人工）、拒绝码、人工复核标记、逾期预测标签；
作用：基线整体通过率、坏账率、分层样本决策分布统计基准。

# 完整执行流程
Step1：读取均衡评测集、基线版本信息、特征/规则/模型清单、决策逻辑；
Step2：生成快照存储表建表语句，包含五层快照全字段、联合唯一索引（版本号+样本ID）；
Step3：逐条绑定样本与基线版本，模拟完整链路生成五层快照结构化数据；
Step4：按批次分片组装批量INSERT入库SQL，配置单批插入条数；
Step5：附加快照数据一致性校验SQL，用于入库后校验快照完整性；
Step6：输出建表语句、批量入库脚本、基线快照管理使用说明。

# 固定输出四段结构（严格顺序不可调换）
## 一、基线快照存储表建表SQL
完整五层快照字段、索引、注释、存储引擎配置
## 二、批量评测样本快照入库分片INSERT脚本
按批次拆分，每条样本携带完整五层链路快照数据
## 三、快照入库后一致性校验SQL
校验样本无缺失、版本数据完整、无重复快照
## 四、基线快照管理使用说明
多版本快照区分、横向对比口径、快照数据查询示例

# 禁止行为清单
1. 禁止缺失五层链路任意一层快照采集逻辑；
2. 禁止快照不绑定基线版本号，无法区分多轮基线；
3. 禁止不分片批量插入，输出超长单条SQL；
4. 禁止不提供快照校验SQL，无法验证入库完整性；
5. 禁止英文为主描述，表注释、字段注释、使用说明全部中文。
